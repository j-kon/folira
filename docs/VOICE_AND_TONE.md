# Folira Voice & Tone Guidelines

Folira speaks to users with a calm, clear, respectful, and human voice. Because Folira is a personal sanctuary for reading, all microcopy should reduce anxiety and avoid technical jargon.

---

## Core Voice Principles

* **Calm**: Never alarm the user. Frame information gently and construct solutions instead of warnings.
* **Clear**: Use simple, direct language. Avoid internal technical terms (like IndexedDB, Blob, Service Worker, or QuotaExceededError).
* **Privacy-Conscious**: Reinforce local data ownership without being overly defensive.
* **Human**: Write as a thoughtful librarian or personal assistant.

---

## Writing Examples & Microcopy Transformations

| Instead of (Technical / Alarming) | Use (Folira Voice) |
| :--- | :--- |
| *"IndexedDB quota exceeded."* | *"Folira does not have enough device storage to save this document."* |
| *"Network unavailable."* | *"You're offline. Your saved documents are still available."* |
| *"Delete record?"* | *"Remove this document from Folira?"* |
| *"Corrupted Blob stream."* | *"Folira could not open this document. The file data may be damaged."* |
| *"Service Worker updated."* | *"A new version of Folira is ready."* |
| *"Enter valid integer."* | *"Please enter a page number between 1 and {total}."* |

---

## Form & Dialog Guidance

* **Confirmation Dialogs**: State clearly what action will be taken and offer a primary positive action ("Remove Document") and a non-destructive cancel option ("Keep Document").
* **Empty States**: Pair a gentle description of what is missing with a single helpful next action.
